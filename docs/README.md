# 📄 CIGA 策划导航

## 🎯 核心设计原则 — 四柱检验

任何策划决策都必须通过这四根柱子的检验：

| 原则 | 核心问题 |
|------|---------|
| 🟢 **低认知压力** | 新人看一局就能上手吗？ |
| 🔵 **可迭代** | 砍掉一半功能，剩下的一半仍然好玩吗？ |
| 🟡 **好玩** | 去掉美术音效，核心循环本身有趣吗？ |
| 🟣 **循环性** | 连续玩 5 局，体验有明显不同吗？ |

**柱间优先级**：`低认知压力 > 好玩 > 可迭代 > 循环性`

> 详见：[四柱检验参考](references/four-pillars.md)
> 当原则冲突时，按此优先级判断（例外：多人竞技游戏将「可迭代」提到「好玩」之前）

## 关联技能

> Claude Code 中调用 `/game-design-coach`，输入你的游戏想法，获得设计评审、头脑风暴或深化建议。

| 技能 | 说明 |
|------|------|
| **game-design-coach** | 🎮 主技能。分析游戏设计，评审机制风险，提供方向建议 |
| **grill-me** | 🔥 可选辅助。需要时手工调用，用来压测某个具体决策 |
| **skill-creator** | 🔧 修改工具。想调整 skill 行为时使用 |

---

## 快速开始

```
/game-design-coach 我想做个海盗主题的 2 人卡牌对战，48h jam
```

Skill 会自动判断场景：
- **出点子** → 发散方向，推荐机制组合
- **评价设计** → 四柱分析，指出风险与改进方向
- **深化设计** → 帮模糊想法细化成可落地方案
- **快速诊断**（48h 场景）→ 只给最危险的问题 + 一个立即方向，不做完整评审

### 典型节奏

```
/game-design-coach <你的游戏想法>
```

几个来回后，如果已有清晰方案，可以切换到:
```
/writing-plans <方案> — 生成 TDD 实现计划
```
