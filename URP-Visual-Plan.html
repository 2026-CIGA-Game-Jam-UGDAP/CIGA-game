🔧 实施计划

1 URP 管线迁移 ~30 min

  1. 安装 URP 包 — Window → Package Manager → Universal RP → Install (版本匹配 Unity 2022.3.62f3)
  2. 创建 URP Asset — Assets/Settings/UniversalRP.asset（右键 → Create → Rendering → URP Asset → 选 2D Renderer）
  3. 创建 2D Renderer Data — Assets/Settings/2DRenderer.asset
  4. 指派管线 — Project Settings → Graphics → Scriptable Render Pipeline Settings → 拖入 URP Asset
  5. 材质升级 — Edit → Render Pipeline → URP → Upgrade Materials → 选 Sprite-Lit-Default
  6. 逐场景验证 — 打开 Level1~4 + Lobby，确认渲染无异常
⚠️ 注意：URP 2D Renderer 使用 2D Light 而非 3D Light。如果场景中有 Point Light / Directional Light，需要替换为 Light 2D 组件。Sprite-Default 材质升级后可能需要手动调整部分素材的材质的 Base Color。

3 星球大气光晕 ~30 min

原理：利用 URP Bloom 的 Threshold 机制——在星球背后放一个白色渐变圆形 Sprite，其亮度超过 Bloom Threshold (0.8)，自动被 Bloom "点燃" 产生辉光。不需要写 shader。


  1. 制作光晕 Sprite — 在 PS/Aseprite 中画一个径向渐变的白色圆形（中心白→边缘透明），256×256 即可。保存到 Assets/Art/VFX/PlanetGlow.png
  2. 挂载到星球 — 在星球 Prefab 下添加子 GameObject Glow，SpriteRenderer 使用这个光晕贴图，Scale 设为星球大小的 1.4~1.6 倍
  3. 调整颜色 — SpriteRenderer.Color 设白色或微彩色（橙色星球用暖黄光晕 #FFEEDD，蓝色星球用青蓝光晕 #CCEEFF），Bloom 会自然地给不同颜色加不同辉光
  4. 排序层级 — Glow 的 Sorting Order 设为星球 -1（在星球后面），让光晕从星球边缘溢出

Circle (星球 Prefab)
├── Glow (SpriteRenderer, 1.5× scale, 白色径向渐变, Order -1) ← Bloom 点燃它
├── SpriteRenderer (星球贴图, Order 0)
├── AnchorPoint
└── CircleCollider2D

📐 参数速查表

Bloom 参数含义

Threshold亮度阈值。低于此值的像素不发光。0.8 = 只有很亮的东西才辉光
Intensity辉光强度。0.5~0.8 适合奇幻但不刺眼
Scatter散射扩散。低=光晕紧贴物体边缘，高=大面积柔光。0.7 = 柔和不失形

视觉效果清单

  • 星球光晕在暗背景上明显可见
  • Bloom 不糊到角色身上
  • 暗角自然不突兀
  • 色彩冷暖对比有层次
  • 场景切换时色调过渡不跳变

  1. 打开 Level1，选中 PostProcessVolume
  2. 在 Inspector 里展开 Volume Profile，逐个调：
    Bloom → Threshold：控制什么亮度才发光（0.5~0.9）
    Bloom → Intensity：辉光有多强（0.3~1.0）
    Color Adjustments → Color Filter：场景主色调
    Vignette → Intensity：暗角深浅（0.2~0.5）
  3. 跑起来看效果，边看边调

Step 3 — 逐场景验证

  1. 依次打开 Level1 → Level2 → Level3 → Level4 → Lobby
  2. 检查无材质丢失（粉色方块）、无渲染异常
  3. 各自微调 Volume Profile 的 Color Filter

Step 4 — 测试陨石闪白

  1. 打开 Level3（有陨石的场景）
  2. 运行，让陨石砸中玩家

🎮 陨石闪白调用方式

已在 Meteor.cs 的 OnTriggerEnter2D 中自动调用。其他地方要用：


PostProcessController.Flash(4f, 0.15f);

参数 1 = Bloom Intensity（默认 4），参数 2 = 持续时间秒（默认 0.15）。


其他地方想加闪白：

  • 爆炸 → PostProcessController.Flash(6f, 0.2f);
  • 轻微碰撞 → PostProcessController.Flash(2f, 0.08f);
  • 通关 → PostProcessController.Flash(3f, 0.5f);

🔗 技术笔记

为什么非迁移 URP 不可？

Built-in 管线下做 Bloom/Color Grading 需要第三方插件（如 Post Processing Stack v2），但 Unity 官方已不再维护它。URP 内建 Volume + Bloom/Color Adjustments/Vignette，零额外安装，配置即生效，且和 2D Renderer 深度集成。

Bloom Threshold 和星球光晕的关系

核心技巧：光晕 Sprite 的顶点色/贴图亮度必须 > Bloom Threshold 才会发光。
具体操作：使用 Sprite-Unlit-Default 材质（不受光照影响，始终全亮输出），光晕 Sprite 设置为白色渐变，亮度超过 Threshold 自动被 Bloom "点燃"。

闪白架构

PostProcessController 在每个场景的 PostProcessVolume 上，维护一个独立的子 Volume（_FlashVolume）专门用于临时效果。闪白时激活该 Volume，用 Bloom Intensity 动画从高值（如 4）快速 lerp 到 0。闪白 Volume 的 Bloom Threshold = 0（所有像素都发光），与全局 Bloom（Threshold 0.8，只让亮的发光）互补。
