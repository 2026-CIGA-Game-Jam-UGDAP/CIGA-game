1. 设计决策汇总

决策点结论
UI 风格混合式：中等立绘 + 对话框占主区域
Hierarchy全局单例 DialogueManager，挂在场景 Canvas 上
触发方式靠近 NPC 进入 Trigger + 按交互键
交互键默认 F（P1 已用 E 传能量），Inspector 可改
对话数据ScriptableObject（DialogueSO），多段顺序循环
对话结构多角色对话，左右对站立绘
每行字段角色名 + 台词 + 立绘 Sprite + 左右站位
动画风格柔和过渡（DOTween 淡入淡出 0.3s）
打字机TextAnimator (Febucci) TypewriterByCharacter，Bool 开关
推进方式按空格/F → 先完成当前打字机 → 再按 → 下一句
输入禁用时停（Time.timeScale=0）+ PlayerController 检查 DialogueManager.IsActive
玩家位置对话中锁位置，禁全部输入（移动/攻击/跳跃）
网络暂无，预留。后续用 Mirror Rpc 广播对话状态
立绘 Idle暂不做，需美术资源

3. 数据流 & 生命周期

玩家走进 NPC Trigger
        │
        ▼
  DialogueTrigger 显示 Hint UI（可选）
        │
        ▼ 按 F
  DialogueManager.StartDialogue(DialogueSO)
        │
        ├── Time.timeScale = 0          ← 游戏时停
        ├── dialoguePanel.SetActive(true)
        ├── PlayerController 检测 IsActive → return（禁用输入）
        │
        ▼
  ShowLine(lines[0])
        ├── UpdatePortraits()           ← 立绘显隐 + 可选 DOTween 淡入
        ├── speakerNameText.text = ...
        ├── useTypingEffect?
        │     ├── true:  typewriter.ShowText() → 逐字打出
        │     └── false: dialogueText.text = text（即时显示）
        │
        ▼
  打字完成 / 即时显示 → OnTypingComplete()
        ├── lineFullyShown = true
        └── pressHint.SetActive(true)   ← "按 空格/F 继续"
        │
        ▼ 按 空格/F
  如果 !lineFullyShown → CompleteTyping()（SkipTypewriter）
  如果 lineFullyShown  → AdvanceDialogue()
        │
        ├── currentLineIndex++
        ├── 如果还有行 → ShowLine(next)
        └── 如果没有   → EndDialogue()
                              ├── 停掉打字机
                              ├── panel.SetActive(false)
                              ├── 立绘复位
                              └── Time.timeScale = 1

4. 场景搭建步骤

4.1 创建对话 UI Canvas

在场景 Hierarchy 中创建（不是代码生成）：

Canvas (命名为 DialogueCanvas，Render Mode = Screen Space - Overlay)
├── DialoguePanel                        ← Image，全屏半透明遮罩，初始 SetActive = false
│   ├── LeftPortraitGo                     ← 空 GameObject（定位容器），初始 SetActive = false
│   │   └── LeftPortrait (Image)            ← 立绘图，Anchor 左中
│   ├── RightPortraitGo                    ← 空 GameObject（定位容器），初始 SetActive = false
│   │   └── RightPortrait (Image)           ← 立绘图，Anchor 右中
│   ├── SpeakerName (TMP_Text)            ← 说话人名字，居中偏上
│   ├── DialogueText (TMP_Text)            ← 对话正文，底部对话框区域
│   │   └── [挂] TextAnimator_TMP           ← Febucci 组件
│   │   └── [挂] TypewriterByCharacter      ← Febucci 组件
│   └── PressHint (TMP_Text)              ← 写 "按 空格/F 继续 ▶"，初始 SetActive = false

4.3 配置 TextAnimator（打字机）

  1. 选中 DialogueText（TMP_Text GameObject）
  2. Add Component → Text Animator - TextMeshPro
  3. Add Component → Typewriter By Character
  4. 推荐参数：
    • Wait For Normal Chars = 0.03
    • Wait Long (! ? .) = 0.6
    • Wait Middle (; : ) - ,) = 0.2

4.4 创建对话数据 (DialogueSO)

  1. Project 窗口右键 → Create → GameJam → Dialogue
  2. 命名如 Dialogue_NPC001_Intro
  3. 填入 Lines 数组：
    • Speaker Name：角色名
    • Text：对话文本（支持 TextArea 多行）
    • Portrait：拖入立绘 Sprite
    • Is Left Side：true=左站位，false=右站位
  4. 迭代开关（默认 false，后续勾选）：
    • Use Typing Effect：启用 TextAnimator 逐字效果
    • Use Animation：启用 DOTween 立绘淡入动画

4.5 配置 NPC + DialogueTrigger

  1. NPC GameObject 上：
    • 确保有 Collider2D，勾选 Is Trigger = true
    • Add Component → DialogueTrigger
    • Dialogues 数组拖入对应的 DialogueSO（可拖多个，按顺序循环）
    • Interact Key 默认 F（Inspector 可改）
  2. 可选：创建一个 World Space 气泡 UI 作为 Hint UI（"按 F 对话"），拖到 DialogueTrigger 的 Hint UI 字段
  3. 确保 Player 的 Tag = "Player"

6. 注意事项 & 已知问题

⚠️ 交互键冲突

  • P1 的 E 键已用于能量传输（PlayerController line 108）
  • 对话触发默认用 F，推进用 空格 或 F
  • Inspector 里 DialogueTrigger.interactKey 可随时改

⚠️ Time.timeScale = 0 影响

  • Update() 仍然运行，但 Time.deltaTime = 0
  • FixedUpdate() 中 Time.fixedDeltaTime = 0，物理停止
  • DOTween 动画需加 .SetUpdate(true) 才能在时停中运行（DialogueManager 已处理）
  • AudioSource 不受 timeScale 影响（如需暂停音效，额外处理）

⚠️ IDE 报错 "DialogueManager 不存在"

  • 新脚本首次创建后 IDE 暂未编译 —— 切到 Unity 等它 Refresh/Compile 后自动消失
  • 如果编译后仍报错，检查文件名与类名是否一致

⚠️ 玩家 Tag

  • DialogueTrigger 用 CompareTag("Player") 识别玩家
  • 确保 Player GameObject 的 Tag 设为 Player
