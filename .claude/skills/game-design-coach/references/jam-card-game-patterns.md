# 🃏 卡牌游戏设计模式参考

> 读取此文件的场景：需要卡牌机制灵感、平衡性参考、或者拆解对标游戏时。

---

## 一、三国杀机制拆解

### 核心系统
```
牌堆 (108张) → 手牌 (每人4张起始) → 出牌阶段 → 目标选择 → 响应链 → 结算
                                                              ↓
                                                       弃牌/装备/技能
```

### 卡牌构成（标准版）
| 类别 | 占比 | 示例 | 功能 |
|------|------|------|------|
| 基本牌 | ~50% | 杀(30)、闪(15)、桃(8) | 攻击/防御/回复 |
| 锦囊牌 | ~40% | 过河拆桥(6)、顺手牵羊(5)、无中生有(4) | 策略/干扰/辅助 |
| 装备牌 | ~10% | 青龙偃月刀、八卦阵 | 持久状态修改 |

### 认知负担分析
```
三国杀的真实认知负担（你自己的经验也回想一下）：

- 🟢 基本规则（每回合摸1出1）：★★☆☆☆ 简单
- 🟡 响应链（杀→闪→桃）：★★★☆☆ 需要学习
- 🔴 规则优先级/时机（"一回合一次" vs "每阶段一次"）：★★★★☆ 常搞混
- 🔴 角色技能差异（100+武将）：★★★★★ 新手根本记不住

对于 48h jam → 模仿前两层，尽量别碰后两层。
```

### 三国杀的"好玩"秘密
1. **社交推理（身份局）**：你不知道谁是队友 → 口嗨 + 演技
2. **翻盘感**：一张牌逆转局势 → 紧张刺激
3. **嘴炮空间**：谈判、欺骗、联盟 — 这是最核心的乐趣
4. **不对称性**：不同武将有完全不同玩法

## 二、48h 可用的卡牌游戏模式

### 模式 1：死斗模式（Free-for-All）
```
最适合 48h jam 的模式

规则：
- 3-4 人各自为战
- 最后存活者获胜
- 没有阵营、没有联盟机制（玩家的私下联盟不管）

优点：
- 最简单的胜利条件判定
- 没有复杂的人际关系推理
- 每个人都是玩家的敌人 → 节奏自然紧凑

```

### 模式 2：组队死斗（Team Deathmatch）
```
规则：
- 2v2 或 3v3
- 队伍共享生命值或各自计算
- 全灭对方队伍获胜

优点：
- 队友配合增加深度
- 每局有战术空间

注意：
- 需要基础队友通信
- 队友死亡后不能"坐牢"——要让死人也有事做

```

### 模式 3：夺旗/Boss 战（Objective-based）
```
规则：
- 所有玩家抢夺一个公共目标（BOSS/宝物）
- 对目标造成最后一击者获胜
- 或：持有宝物一定回合者获胜

优点：
- 目标导向 → 玩家自然知道该做什么
- 动态变化 → 领先者成为众矢之的

```

## 三、卡牌类型设计模式

### 基础三件套（推荐 48h 起步用）

```
进攻牌 → 对目标造成伤害         → 如："飞刀" 造成 1 点伤害
防御牌 → 抵消伤害或减少伤害     → 如："闪避" 抵消 1 点伤害
恢复牌 → 回复生命值或资源       → 如："丹药" 回复 2 点生命

这就是三国杀的"杀/闪/桃"模型。简单、有效、经过市场验证。
```

### 策略牌设计（Iteration 1 加入）

| 效果类型 | 例子 | 设计要点 |
|---------|------|---------|
| 强制性 | "目标弃 1 张牌" | 最直接的干扰 |
| 资源类 | "多摸 2 张牌" | 增加手牌优势 |
| 转移类 | "把一张牌移给其他人" | 创造社交互动 |
| 条件类 | "如果手牌最少，摸 3 张" | 补偿机制（落后友好） |

### 装备牌 — 建议只在 Iteration 2 引入

装备系统的复杂度：
```
加上装备前：伤害判定数值，简单
加上装备后：攻击距离计算、装备类型冲突、破甲/防甲 优先级
                  ↓
           需要装备栏UI（武器/防具/坐骑...）
                  ↓
           需要装备切换和替换逻辑
                  ↓
           需要网络同步装备状态
```

如果决定做装备，保持**一件装备、一个插槽**的简单模式。

## 四、不对称角色设计

### 不对称程度光谱

```
轻度（推荐入门）          中度（推荐进阶）        重度（不推荐 48h）
─────────────────────────────────────────────────────────────
血量不同                 独特技能               完全不同的牌组
起始手牌不同             资源机制不同           不同的游戏目标
                         限制性能力            不同的获胜方式

示例：                    示例：                 示例：
A：4血 3起始手牌          A：每回合多摸1张        A：要收集4件宝物获胜
B：3血 4起始手牌          B：受伤可摸1张          B：要击败所有怪物获胜
C：4血 2起始手牌 + 1装备  C：可用血换牌          C：要存活到10回合
```

### 48h 推荐的不对称设计法则

1. **差异在"量"不在"质"**：血量不同、手牌数不同（数值差异）远比"你有飞行能力"这种质变好实现
2. **一个独特能力就够了**：给每个角色加 1 个独特被动效果。不要在角色上堆砌能力
3. **能力要容易被对手理解**："他每回合多摸 1 张" — 对手看到他在摸牌就知道这个能力在生效。"他受伤时可以弃我一张牌" — 对手需要记忆这个能力，认知负担就高了

### 48h 角色设计示例（4 角色）

| 角色 | 血量 | 起始手牌 | 独特能力 | 认知负担 |
|------|------|---------|---------|---------|
| 战士 | 5 | 3 | 无（纯数值优势） | ★ |
| 刺客 | 3 | 4 | 你的进攻牌伤害 +1 | ★★ |
| 医者 | 4 | 3 | 你的恢复牌效果翻倍 | ★★ |
| 谋士 | 3 | 5 | 每回合可额外弃一张牌换摸一张 | ★★★ |

> 医者和谋士的能力虽然都是 ★★~★★★，但它们的效果是"自己生效"的，对手不需要记住复杂规则。

## 五、快速平衡性指南（48h 版）

### 平衡原则：48h 不需要精确平衡
- 你的游戏只会在 jam 中被玩 20-50 次，不需要万分之一平衡
- **"明显太强"比"略强"更致命** — 不要让某个角色/卡牌胜率 > 75%
- **检验方法**：自己玩 3 局，如果同一角色赢了 3 次 → nerf 它

### 简单平衡公式
```
角色强度 ≈ 血量 × 0.3 + 手牌收益 × 0.4 + 特殊能力 × 0.3

示例：
战士 = 5×0.3 + 3×0.4 + 0 = 1.5 + 1.2 = 2.7
刺客 = 3×0.3 + 4×0.4 + 1(伤害+1) = 0.9 + 1.6 + 1 = 3.5 ❌ 太强了

→ 调成 刺客 3血 3手牌 伤害+1 = 0.9+1.2+1 = 3.1 ← 和战士接近了
→ 或者保留 3血4手牌但伤害+1 改为 伤害+1 但每回合少摸1张
```

> 这个公式是粗略估算工具，不是精确科学。但它能帮你避免明显的平衡灾难。

## 六、回合流程模式

### 模式 A：同步回合（推荐）
```
所有玩家同时：
1. 选择一张牌背面朝上打出
2. 所有人亮牌
3. 按顺序或优先级结算
4. 同时进入下一回合

优点：等待时间短，节奏快
技术实现：所有客户端同时提交 Command，服务器统一结算
```

### 模式 B：交替回合（经典）
```
玩家1 → 玩家2 → 玩家3 → 玩家1 → ...

优点：规则直观，类似传统桌游
技术实现：服务器维护 currentPlayer，只接受当前玩家的 Command
```

### 模式 C：速度回合（进阶）
```
阶段1（出牌）：所有玩家同时选择行动，超时视为 PASS
阶段2（结算）：根据速度值或优先级依次结算

这是"同时出牌"和"轮流结算"的混合，
可以创造同时行动的快节奏 + 有序结算的可预测性
```

## 七、上手教程设计

### 48h 不需要教程

真的。48h 不要写教程。用这些代替：

```
✅ 文字提示卡：开局显示 3 条核心规则（UI 文本，无需交互）
✅ 标记式卡牌：卡牌效果写在卡面上（"造成 1 点伤害"）
✅ 默认选中：选中玩家时自动选中手牌第一张
✅ AI 占位：单人测试模式让用户自己操作两个客户端

❌ 不要做：
- 交互式教程/新手指引（要写 tutorial controller，48h 不值得）
- 弹窗规则说明（玩家会直接关掉）
- 视频教程（没时间录也没地方放）
```

## 八、让游戏"好玩"的已知配方

| 配方 | 描述 | 例子 |
|------|------|------|
| **高风险高回报** | 赌一把的爽感 | "弃 1 张牌随机造成 1-3 伤害" |
| **绝地翻盘** | 落后时有翻盘点 | "当生命 ≤ 1 时，摸 2 张牌" |
| **你来我往** | 不是你碾压我 | 攻防节奏交替（这需要防御牌存在） |
| **吹牛/欺骗** | 社交层面的乐趣 | 背牌出牌，别人不知道你出了什么 |
| **团队瞬间** | 队友配合杀 | 2v2 中 A 控制住敌人，B 输出 |

### 48h 最容易做的好玩配方
1. **随机翻牌 + 赌徒心理** — "赌这张牌是好的还是坏的"
2. **落后保护** — 血越少/劣势越大时获得补偿
3. **公共牌堆中的争夺** — 大家都抢同一张好牌
