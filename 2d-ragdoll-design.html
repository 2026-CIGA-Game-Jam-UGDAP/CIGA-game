🎯 2D 力驱动假布娃娃（Force-Driven Fake Ragdoll）

项目：CIGA 太空双人合作 · 俯视角 2D · Unity + Mirror · 2026-07-04

设计目标

在 不拆分 Sprite、不加骨骼、不引入新资源 的前提下，让角色在受击/死亡/弹飞时产生类似 3D 布娃娃的"失控翻滚"效果。

🎬 逐帧效果

以 P1 被星球表面爆炸向右上方弹飞为例：

#0 🧍 竖直站在星球表面，已吸附
#1 💥 DetachFromAnchor() → 解锁 Z 旋转 → AddForce(↗, 大) + AddTorque(CW 500°)
#2 🕴️ 沿右上方飞出，开始顺时针旋转 ~30°
#3 ↷ 旋转加速到 ~120°，身体横过来了
#4 ↻ 旋转到 ~250°，头朝下
#5 🌀 旋转到 ~400°，飞出 ~3 单位
#6-8 angularDrag 减速旋转，linearDrag 衰减飞行
#9 角色几乎停下
#10 🔒 FreezeRotation = true · 恢复姿态
或（死亡）→ 不恢复，歪着漂在太空里

核心机制

三个关键参数

参数值小值大
angularDrag
角阻力		角色像陀螺一直转
搞笑 / 失控感		转半圈就停了
沉重 / 真实感
linearDrag
线阻力		飞很远，太空漂浮感
当前项目值 0.05		飞一小段就停
像在水里被打
torque
初始扭力		翻一圈
像被打一拳		翻五六圈
像被车撞

状态机

正常飞行（FreezeRotation=true）
  │
  ├── 受击/弹飞
  │     ├── DetachFromAnchor()
  │     ├── FreezeRotation = false
  │     ├── AddForce(direction × force)
  │     └── AddTorque(random ±360~720°)
  │
  ├── 翻滚中（angularDrag + linearDrag 自然衰减）
  │
  └── 恢复判定
        ├── 存活 → 1.5s 后 FreezeRotation=true，可喷气恢复
        └── 死亡 → 不恢复旋转，就这样漂着

应用场景

场景 A：碰碰车弹开升级

现在：两人相撞 → surfaceT ±0.4 → 表面滑开 → 全程竖直 → 碰碰车感 ✓

升级后：两人相撞 → Detach + 解锁旋转 + 各自弹飞 + 空中翻滚 → 1s 恢复

碰前：🧍→  ←🧍 星球上相向而行
碰后：🧍↗️💫 左边弹出 + 逆时针翻两圈 + 飞离星球
      🧍↖️💫 右边反向弹出 + 顺时针翻一圈半
1s后：各自喷气恢复姿态，重新飞回星球

场景 B：被流星/障碍物击中

击中瞬间：猛烈旋转 + 沿撞击方向飞出
         + SpriteFlash（已有组件）
         + CameraShake（已有组件）
0.5s后：  旋转减速 + 飞行衰减 → 角色歪着漂
恢复后：  FreezeRotation + 可喷气返回

场景 C：死亡

死亡时不恢复旋转 → 歪着/倒着漂在太空里
缓慢自转残留（角速度不归零）
sprite 变灰/变暗 → 视觉明确是「尸体」

备选方案对比

方案成本效果适合
方案 2 力驱动 十几行代码 角色整体翻滚飞出的物理感 ⭐ 本项目首选
方案 1 分离肢体 美术切图 + 关节配置 头/身体/四肢各自飞，真正布娃娃 有美术资源时升级
方案 3 精灵动画 美术逐帧 死亡动画固定，不动态 无物理需求时
插件 Puppet2D / 2D Animation 骨骼绑定 + 插件学习 需要骨骼系统 ❌ 本项目用不到

实现要点

  • 在 PlayerController 中新增 Knockback(Vector2 direction, float force) 方法
  • 调用时自动 DetachFromAnchor() 解除吸附
  • 解锁 RigidbodyConstraints2D.FreezeRotation
  • 击飞后通过协程在 1~2 秒后恢复旋转锁定（存活时）
  • 死亡时不恢复，角色歪着漂
  • Mirror 联网：击飞由服务器触发 → RPC 广播力方向和力度 → 各客户端本地模拟物理

一句话总结

角色从"永远竖直的棋子"变成"会翻滚、会被打飞、有重量感的物理物体"，不需要拆零件也不增加任何资源。代价只有 4 个参数 + 十几行代码。
