1. 摄像头抖动 — CameraShake

文件Assets/Scripts/CameraShake.cs
触发方式cameraShake.Shake() / Shake(float intensity)

Editor 配置

  1. 在 Player Prefab 上添加组件 CameraShake
  2. 在 Player 下建子对象 CameraShakeSource，挂 CinemachineImpulseSource
  3. 把 CameraShakeSource 拖到 CameraShake 的 impulseSource 字段
  4. 在 Main Camera（两个场景）添加 CinemachineImpulseListener
    Gain = 1（越大越抖），Channel = 1（默认）
  5. 在 Player 下建 CinemachineVirtualCamera（使 CinemachineBrain 有跟随目标）
  6. 把 ImpulseListener 放到 VirtualCamera 上（注意：ImpulseListener 在 CM 2.x 中需要放在 VirtualCamera 上才能产生抖动效果）

Inspector 参数

参数说明建议值
impulseSourceCinemachineImpulseSource 引用拖子对象
defaultIntensityShake() 无参时的默认强度1.0
ImpulseSource 上调Signal Shape、Duration、衰减等Shape=Random, Velocity=(1,1,0)
💡 调手感：ImpulseSource 的 Default Velocity 决定抖动方向。
俯视角建议 (1, 1, 0) = XY 平面随机抖，去掉 Z。Impulse Duration 长=晃得久。

2. 弧形受伤指示 — DamageIndicator

文件Assets/Scripts/DamageIndicator.cs
触发方式damageIndicator.Show(Vector3 worldDirection)

Editor 配置

  1. 在 GamePlay 场景新建 Canvas（Screen Space - Overlay）
  2. 在 Canvas 下新建 UI Image，命名 DamageArc
  3. Anchors = stretch/stretch（填满画布），Pivot = (0.5, 0.5)
  4. 把一张 弧形红光 Sprite（90° 弧形半透明）拖到 Image 的 Source Image
  5. Image 的 Color.Alpha 初始设为 0
  6. 在 Player Prefab 上添加 DamageIndicator
  7. 把场景中的 DamageArc Image 拖到 arcImage 字段（建议将 DamageArc Image 拖拽到 Player prefab 上转为预制体引用，避免场景引用丢失）

Inspector 参数

参数说明建议值
arcImage弧形 UI Image 引用必须拖
showDuration全透明度保持时间(秒)0.5
fadeDuration淡出时间(秒)0.3
maxAlpha最大透明度0.7
🎨 弧形 Sprite 怎么做？
最快：Photoshop/GIMP 画一个 90° 扇形/弧形，红色半透明渐变（边缘亮中心淡），导出 PNG 导入 Unity。
没有图也可以先用一张红色矩形 Image ——旋转后虽然不圆但仍能指示方向，后续替换 Sprite 即可。

3. 地面冲击波 — Shockwave

文件Assets/Scripts/Shockwave.cs
触发方式Shockwave.Play(prefab, worldPos) 静态方法

Editor 配置

  1. 建空 GameObject，命名 Shockwave
  2. 添加 SpriteRenderer，挂一张 圆环 Sprite（空心圆/径向渐变）
  3. 添加 Shockwave 组件
  4. 保存为 Prefab 到 Assets/Prefabs/Effects/Shockwave.prefab
  5. 把 Prefab 拖到 PlayerController 的 shockwavePrefab 字段

Inspector 参数

参数说明建议值
startRadius初始缩放（localScale）0.2
targetRadius最大缩放（localScale）5.0
expandDuration扩散阶段持续时间0.4
fadeDuration淡出阶段持续时间0.3
⚠ Sprite 尺寸注意：shockwave 的缩放 = localScale，不是世界单位。
假设 Sprite 的 PPU=100，宽 100px=1 世界单位，targetRadius=5 则最终直径约 5 单位。
如果 Sprite 原始尺寸不同，调 targetRadius 直到看上去合适。

4. 受击闪白 — SpriteFlash

文件Assets/Scripts/SpriteFlash.cs
触发方式spriteFlash.Flash() / Flash(color, count)

Editor 配置

  1. 在 Player Prefab 上添加 SpriteFlash 组件
  2. 不需要拖引用——GetComponentInChildren<Renderer>() 自动找
  3. 把 SpriteFlash 拖到 PlayerController 的 spriteFlash 字段

Inspector 参数

参数说明建议值
flashColor闪烁颜色白色 #FFFFFF
flashDuration每帧闪烁周期(秒)0.1
flashCount闪烁次数1（闪一次）
💡 SyncVar 注意：代码临时改 material.color 和 SyncVar 的 playerColor 互不干扰。
但若 SyncVar hook 在闪白期间触发，颜色会立即跳到 SyncVar 的值（中断闪白）。
48h Jam 里碰到的概率极低，忽略即可。

5. 弹性出生 — SpawnBounce

文件Assets/Scripts/SpawnBounce.cs
触发方式spawnBounce.Play()（已在 OnStartLocalPlayer 中调用）

Editor 配置

  1. 在 Player Prefab 上添加 SpawnBounce 组件
  2. 拖到 PlayerController 的 spawnBounce 字段
  3. 就绪——玩家生成时自动触发弹性出生

Inspector 参数

参数说明建议值
bounceCurve弹性曲线（0→过冲→回弹→1）0→1.15→0.95→1.0
duration动画总时长(秒)0.6
🎮 调手感：
更弹：把曲线第二个点 Y 拉到 1.3~1.4（更大的 overshoot）
更干脆：缩短 duration 到 0.3，曲线改为 0→1→1（直冲无过冲）
没有 AnimationCurve 操作经验：在 Inspector 里双击曲线打开 Curve Editor，鼠标拖点即可。

6. 受伤缩抖 — DamagePulse

文件Assets/Scripts/DamagePulse.cs
触发方式damagePulse.Play()（已在 PlayHitFeedback 中调用）

Editor 配置

  1. 在 Player Prefab 上添加 DamagePulse 组件
  2. 拖到 PlayerController 的 damagePulse 字段
  3. 就绪——每次受击自动触发缩放脉冲

Inspector 参数

参数说明建议值
pulseCurve脉冲曲线（1→膨胀→压缩→回稳）1→1.25→0.9→1.05→1.0
duration动画总时长(秒)0.4
🎮 调手感：
肉感膨胀：默认曲线 (1→1.25→0.9→1.05→1) —— 先胀再缩，像被重击
硬直感：改成 (1→0.9→1.05→1) —— 先压缩再回弹，像被戳
抖动感：加多几个振荡点 (1→1.1→0.95→1.05→0.98→1.0)

🎇 粒子预制体配置

以下 3 个效果使用 Unity Particle System，零代码——按表在 Inspector 里填参数即可。建好存为 Prefab，需要时 Instantiate。

死亡爆散 — DeathBurst

击杀时角色碎片四散，带颜色继承。

模块参数值
MainDuration / Start Lifetime / Start Speed / Start Size / Start Color / Simulation Space / Stop Action 0.3 / 0.3–0.6(随机) / 3–6(随机) / 0.05–0.15(随机) / 随机色或继承 / World / Destroy
EmissionRate / Bursts0 / Count=30, Time=0
ShapeShape / RadiusSphere / 0.2
Size over Lifetime启用 / 曲线✓ / 1→0
Color over Lifetime启用 / Alpha✓ / 1→0
RendererMaterialDefault-Particle

用法：Instantiate(deathBurstPrefab, position, Quaternion.identity)，StopAction=Destroy 自动回收。

移动扬尘 — MoveDust

角色脚下尘烟，"在跑"的视觉证据。零代码，挂上即用。

模块参数值
MainDuration / Start Lifetime / Start Speed / Start Size / Start Color / Simulation Space 0.5 / 0.2–0.4(随机) / 0.5–1.5(随机) / 0.03–0.08(随机) / 灰白(0.8,0.8,0.8,0.5) / World
EmissionRate over Time / Rate over Distance0 / 1
ShapeShape / Radius / Align to DirectionCircle / 0.1 / ✓
Size over Lifetime启用 / 曲线✓ / 1→0.3
Color over Lifetime启用 / Alpha✓ / 0.6→0
RendererMaterialDefault-Particle
💡 关键技巧：把 MoveDust ParticleSystem 放在 Player 的子对象下。
Emission → Rate over Distance = 1 意味着玩家每移动 1 单位距离发射 1 粒尘，
站着不动时自动停喷。不！需！要！任！何！脚！本！

🧩 PlayerController — 效果入口

所有 TA 效果通过 PlayerController 统一接入，两个入口方法：

受击反馈 — 一次调用触发所有效果

/// 调用示例：当玩家受击时
playerController.PlayHitFeedback(attackSourcePosition, shakeIntensity: 1f);

// 内部依次触发：
//   cameraShake.Shake(intensity)       → 抖屏
//   damageIndicator.Show(worldDir)     → 弧形指示
//   spriteFlash.Flash()                → 闪白
//   damagePulse.Play()                 → 缩抖
//   impactLines.Play()                 → 冲击线

冲击波 — 生成地面扩散圆环

/// 调用示例：爆炸/大招时
playerController.PlayShockwave(explosionPosition);

// 内部：
//   Shockwave.Play(shockwavePrefab, position)
//   → Instantiate prefab → 自动扩散 + 淡出 + 自毁

Player Inspector 拖引用总表

以下所有字段都在 Player Prefab 的 PlayerController 组件上拖好：

字段类型来源
cameraShakeCameraShake同 GameObject 上的 CameraShake 组件
damageIndicatorDamageIndicator同 GameObject 上的 DamageIndicator 组件
spriteFlashSpriteFlash同 GameObject 上的 SpriteFlash 组件
damagePulseDamagePulse同 GameObject 上的 DamagePulse 组件
impactLinesImpactLines同 GameObject 上的 ImpactLines 组件
spawnBounceSpawnBounce同 GameObject 上的 SpawnBounce 组件
shockwavePrefabGameObjectShockwave 预制体
💡 拖引用技巧：
在 Player Prefab 的 Inspector 中选中 PlayerController，把上述组件从 同一 GameObject 的组件列表拖到对应字段——Unity 允许从组件标签头直接拖拽到 SerializeField 字段。
操作：点击组件名（如 CameraShake）上方标签头 → 拖拽到 PlayerController 的对应字段。

🧪 临时测试代码

在 PlayerController.Update() 的 if (!isLocalPlayer) return; 后面添加：

// T 键：模拟受击（从右侧来）
if (Input.GetKeyDown(KeyCode.T))
    PlayHitFeedback(transform.position + Vector3.right, 1f);

// B 键：模拟冲击波
if (Input.GetKeyDown(KeyCode.B))
    PlayShockwave(transform.position);

运行游戏（按 Host 本地测试），按 T/B 看效果。调参满意后删掉这 4 行。
